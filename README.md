# Joshua J. Root's Portfolio

## Overview
In this portfolio, you will find various applications' codebases that you can not only look at the code for, but that you can download and try for yourself. It is important to keep in mind, however, that for some projects, you will need the appropriate runtimes, DBMS's or other dependencies to make some things work. In this portfolio of projects, I tried to cover the wide range of skills that I posess and put them to a practical use, while using industry standard practices. I also took great care to ensure that each and every application is as accessible as possible for those with disabilities. It is my belief that everyone should be able to use a piece of software without obstacles, even if you have a disability. That belief is the foundation upon which I build any project I work on from the start, instead of it being  an afterthought.

## The ASP.NET Core Applications

Each ASP.NET Core application type has 2 projects associated with them. These projects are built with .NET 7, and will be updated as new .NET versions become available. I will also note here that each project was created using the .NET CLI, not the Visual Studio IDE. Each NuGet package, such as Dapper, Entity Framework Core, and the System.Data namespace were also added and linked to the projects that require them using the .NET CLI, instead of Visual Studio.

## The Windows Forms and WPF Applications
For the Windows Forms and WPF projects, I opted for the approach of creating the same app, but in different user interface frameworks to demonstrate that I can build the same app in a different UI library if I were to ever need to. Much of what is true about the creation of the ASP.NET Core-based projects are also true here. All Windows Forms and WPF projects were made for .NET 7, were created with the .NET CLI, and any dependencies were added and linked to the necessary projects also with the .NET CLI. It is also prudent to say that each app was written in both C# and in Visual Basic.NET.

## The Web Applications

The web projects are in their own category, because projects in this category use packages like Electron to make desktop applications, using web technologies, use a specific browser API, or simply demonstrate that I can build rich and accessible user experiences with just developing a simple website.
